## Motivation: where this comes from

§v2.4-proxy-4b seeded canonical AND bodies into the initial population
at 0.1% / 1% to test the "discoverability-limited" reading of the
proxy-basin-attractor finding. Expected outcome: if selection holds
canonical once found, F high ⇒ R high.

Observed: F=20/20 (every run keeps a canonical elite) but
R_exact ≤ 0.036 (canonical fails to propagate across the population).
§v2.4-proxy-4c replicated the dissociation across three decoder/executor
cells.

**Mechanism reframe (revised 2026-04-17 post-§v2.4-proxy-4d).** The
canonical body is a point attractor for the elite slot but not a basin
attractor for the population distribution. Mutation erodes canonical
out of non-elite slots faster than selection can propagate it. The
**why** differs across decoder arms:

- **Arm A (direct GP):** the canonical plateau is genuinely narrow.
  §v2.4-proxy-4d measured R_fit(≥0.999) = 0.004 at sf=0.01 — only the
  elite solves; the rest of the population sits in the proxy basin
  (mean fitness 0.836, matching classical proxy-basin-attractor). Few
  non-canonical solvers exist; mutational load lands mostly off-plateau.
- **BP_TOPK (preserve + consume):** a *wide* solver neutral network
  exists but canonical is off-center within it. §v2.4-proxy-4d measured
  R_fit(≥0.999) ≈ 0.72 — 72% of the final population solves at fitness
  ≥ 0.999, yet only 0.2% is canonical-adjacent (R₂ ≤ 0.002). The
  chem-tape decoder's many-to-one genotype→phenotype mapping creates a
  large task-solving plateau. Mutation drifts the population *laterally
  across the solver network* rather than *off* the plateau.

So the "mutational load lands mostly off-plateau" reading holds for
Arm A but is wrong for BP_TOPK. BP_TOPK's mutational load lands on
*other solvers* — a different problem that plasticity does not
obviously address, because the plateau already exists; it just is not
centered on canonical.

## Why plasticity (vs the other plateau-response options)

The plateau response space includes:
- Selection-level: ranking / (µ,λ) / Pareto / lexicase
- Variation-level: bond_protection ratios, repair mutations, mutation-rate
  sweeps
- Representation-level: AutoMap-style denoising autoencoder
- Descriptor-based QD: MAP-Elites with hand-picked or learned (AURORA)
  descriptors
- Coevolution: POET, MCC

All descriptor-based and QD approaches *shift* the "what counts as
diversity" choice to a different layer (encoder architecture,
co-population structure, information-theoretic prior); they don't
eliminate it.

**Runtime plasticity is the one option that does not require a
descriptor.** The gradient is generated by the adaptation dynamic itself:
amount-of-adaptation-work-needed-to-reach-canonical becomes the implicit
gradient across the plateau. Hinton & Nowlan 1987 ("How Learning Can
Guide Evolution") is the canonical proof that this works even when the
underlying landscape is flat (needle-in-haystack).

This is also *structurally aligned with the folding thesis.* The
developmental mapping already does one layer of smoothing; plasticity
adds a second at evaluation time. The evolved genotype encodes a
learning algorithm, not a final answer — Schmidhuber's "evolving
learning rules" (1987→) is the ancestry; transformer in-context learning
is a rediscovery of the same structure in a different substrate.

## Structural frame

Current evaluation:
```
genotype → decode → phenotype P → fitness f(P)
```

Runtime-plasticity evaluation:
```
genotype → decode → phenotype P₀ with state
  → train phase: P sees (x₁, y₁) … (x_k, y_k), updates internal state
  → test  phase: P sees held-out x, outputs ŷ → fitness f(ŷ)
```

## Shortcuts ranked by compute cost (pre-GPU)

| Rank | Mechanism | Layer | Rust change |
|---|---|---|---|
| 1 | **Operator-threshold plasticity** — tunable threshold per operator, adapts on train rows, frozen for test | Execution | Localized (one float per op) |
| 2 | **Small persistent memory** — 4–8 bit register on the circuit; two-phase eval forces genotype to encode a write-rule | Execution | Moderate (memory field + read/write ops) |
| 3 | **Hebbian plastic bonds** — bond strengths adjust under co-activation during training phase | Execution | Moderate (per-bond strength state) |
| 4 | **Reservoir split** — evolved part frozen, small linear readout trains per evaluation | Execution | Moderate (readout layer) |
| 5 | **Self-modifying tape** — circuit writes to its own tape during execution | Execution | Large; degenerate-behavior risk high |
| 6 | **Decoder-phase feedback** — chem-tape decoder reads a partial-fitness signal during folding and branches | Development | Large; closest to biological plasticity but deep rewrite |

Start with rank 1. It is the cheapest probe that still tests the
Baldwin hypothesis.

## Critical diagnostic (must be in the prereg)

Two outcomes must be distinguishable, otherwise the experiment is
non-falsifiable:

- **Baldwin outcome:** adaptation gap scales with genotypic distance to
  canonical → the genotype carries real signal; selection pressure on
  genotypes is preserved through the adaptation wrapper.
- **Universal-adapter outcome:** all genotypes converge to similar
  adapted phenotypes regardless of starting point → plasticity is doing
  all the work; the genotypic signal is hollowed out; evolution has no
  grip.

**Diagnostic plot:** frozen fitness vs plastic fitness, both against
Hamming-to-canonical. Baldwin = monotone relationship (closer genotypes
adapt better). Universal adapter = flat plastic-fitness curve regardless
of distance.

This must be a pre-committed outcome in any prereg, with cell entries
for both axes. (Methodology §2 — the §v2.4-proxy-4b outcome-table
incompleteness failure is the cautionary tale.)

## Task-design warning

AND has only 4 truth-table rows — memorizable by a 4-bit lookup. A
sufficiently expressive plastic operator will trivialize fitness without
testing the hypothesis. Two escapes:

1. **Hold-out split**: train on 3 rows, test on the held-out row.
   Forces generalization. Cheapest fix.
2. **Meta-learning framing**: evolve circuits that learn *arbitrary*
   2-bit boolean functions from 3 examples. Population now evolves
   general learners rather than AND-specific circuits.

Framing (2) is the bigger research swing. It sidesteps the AND-specific
plateau entirely and stress-tests Baldwin-vs-universal-adapter harder
(because a universal adapter for arbitrary 2-bit functions is a much
less trivial artifact — if it evolves, that is itself a headline
finding). This connects to the Part-1 meta-learning direction in
`future-experiments.md` — both are moves toward evolving developmental
systems that *learn*, but from different angles (outer-loop ES vs
within-lifetime adaptation).

## Suggested first experiment

**§v2.5-plasticity-1a (hypothetical, not yet pre-registered) — scoped
to Arm A post-§v2.4-proxy-4d:**

- Mechanism: rank-1 shortcut — operator-threshold plasticity.
- **Decoder arm: direct GP (`arm='A'`), not BP_TOPK.** §v2.4-proxy-4d
  showed BP_TOPK already produces a wide solver plateau via its
  many-to-one decode (R_fit ≈ 0.72). Plasticity on top would be
  redundant and would confound interpretation — we could not tell
  whether any R_fit lift came from plasticity or from underlying
  decoder smoothing. Arm A has a narrow plateau (R_fit = 0.004);
  plasticity has unambiguous work to do there.
- Task: AND with 3-row train / 1-row test hold-out.
- Pop=512, gens=500, N=20 seeds.
- M1 CPU adequate; no GPU needed.
- Measure: F on held-out row, R_exact, **R_fit(≥0.999)** (the 4d
  diagnostic that surfaces solver-network width), *and* the
  frozen-vs-plastic gap stratified by Hamming-to-canonical.
- Outcome table (must pre-commit) — principle 2b-compliant grid across
  (F-recovery, Baldwin-diagnostic) axes:
  - PASS: F recovers on held-out row AND frozen-vs-plastic gap
    correlates with Hamming-to-canonical (Baldwin).
  - PARTIAL: F recovers but gap is flat (universal adapter —
    plasticity works but dissolves evolutionary signal).
  - FAIL: F does not recover (plasticity too weak / landscape too hard).
  - INCONCLUSIVE: any outcome not in the above cells.

If PASS: (1) escalate to memory (rank 2), then to meta-learning framing;
(2) **run the same probe on BP_TOPK as a control cell** — does
plasticity add R_fit lift on top of the already-wide solver plateau,
or is it additive / redundant? The differential magnitude across
decoder arms is itself a finding about whether plasticity and
structural decoder smoothing are substitutes (interchangeable) or
complements (operate on different layers).
If PARTIAL: tighten the adaptation budget until the gap re-appears,
or redesign the plastic substrate.
If FAIL: try higher adaptation capacity before concluding.

## Open questions / future work

- **Adaptation budget as a control parameter.** Too little budget = no
  Baldwin. Too much = universal adapter. There should be a sweet spot,
  and *where* it lies is itself informative about landscape structure.
- **Does plasticity rescue the scaffold result? (Partially answered by
  §v2.4-proxy-4d.)** The population-propagation failure has two
  distinct mechanisms across decoder arms: narrow plateau under Arm A
  (canonical-neighbor fitness cliff), wide solver plateau under BP_TOPK
  (canonical off-center). Plasticity directly addresses the Arm A
  mechanism. For BP_TOPK, plasticity is likely orthogonal — the plateau
  is already wide; the question there is *which solver gets centered
  on*, not whether solvers are findable. So plasticity is not
  interchangeable with Pareto-scaffold preservation across the board;
  they address different decoder arms.
- **Interaction with meta-learning (Part 1 of future-experiments.md).**
  Runtime plasticity evolves *within-lifetime* learners. ES over
  chemistry params evolves *between-lifetime* learners. Both together
  would be a two-level system (Schmidhuber-style). Do they compose,
  or does one dominate?
- **Relation to "drift as lateral search."** Plasticity creates a
  gradient *on* the plateau, replacing drift with directed motion.
  Does this eliminate the drift-to-portal regime entirely, or does
  drift still help by exposing new regions of the plateau for
  plasticity to exploit?
- **Connection to the folding-track thesis. (Sharpened by §v2.4-proxy-4d.)**
  BP_TOPK's decoder already does substantial plateau smoothing
  (R_fit = 0.72); Arm A does essentially none (R_fit = 0.004). The
  clean comparison design: run the plasticity probe under both arms,
  measure R_fit lift in each. If BP_TOPK sees a small lift and Arm A
  sees a large lift, structural decoder smoothing and lifetime-
  plasticity smoothing are **substitutes** (interchangeable — they do
  the same work at different layers). If both see similar magnitudes,
  they are **complements** (they operate on different layers of the
  landscape and compose). This is a cleaner test of the folding-thesis
  claim than was available before §v2.4-proxy-4d provided the
  per-decoder baselines.

## Why this is worth the compute

The cheap options (bp=0.7 sweep on BP_TOPK, mutation_rate sweep on both
arms) should run first — they directly test variation-layer interventions
on the two distinct failure modes §v2.4-proxy-4d revealed (narrow plateau
under Arm A vs canonical-off-center drift under BP_TOPK). The
tournament-size axis is demoted post-4d: with 72% solvers already present
under BP_TOPK, weakening selection mostly widens drift across the solver
cloud — not informative for the retention question. If the bp/mutation
sweeps lift R₂, plasticity may be unnecessary for the immediate question
under that decoder arm.

But plasticity is the direction even if they *do* lift R, because:
1. It is the one plateau-response that does not require a descriptor.
2. The meta-learning framing connects to evolved in-context learning —
   a much bigger research claim than plateau-smoothing.
3. It is structurally aligned with the folding-evolution thesis
   (developmental systems that learn within lifetime).
4. It connects Part-1 meta-learning work to within-lifetime adaptation,
   potentially giving a unified two-level story.

The first experiment is cheap (M1 CPU). Only if it passes does the
GPU / cloud spend become justified — specifically for the meta-learning
framing at scale.
